## Quickstart

1. Clone repository:

```bash
git clone https://github.com/deseyebags/SBCbank.git
cd SBCbank
```

2. Prepare backend runtime environment:

```bash
cp backend/.env.runtime.example backend/.env.runtime
```

Windows PowerShell alternative:

```powershell
Copy-Item backend\.env.runtime.example backend\.env.runtime
```

3. Start local PostgreSQL and load mock data:

```powershell
./scripts/start-local-postgres-and-load-data.ps1
```

This starts the local `postgres` container and seeds dashboard-ready records into the local `scbbank` database.

4. Start backend stack:

```bash
cd backend
docker compose up --build -d
```

5. Stop backend stack:

```bash
cd backend
docker compose down
```

6. Start frontend development server:

```bash
cd frontend
npm install
npm run dev
```

7. Sign in through the frontend at `http://localhost:5173/login`:

- Admin: `admin` / `admin123`
- User: account ID + matching account email

## Metabase Dashboards (QuickSight Alternative)

Use Metabase locally to build analytics dashboards while QuickSight is unavailable in LocalStack.

1. Start backend stack (now includes Metabase):

```bash
cd backend
docker compose up --build -d
```

2. Open Metabase setup at `http://localhost:3001`.

3. Connect Metabase to PostgreSQL with:

- Host: `postgres` (or `localhost` if connecting from host outside Docker)
- Port: `5432`
- Database: `scbbank`
- Username: `scbbank`
- Password: `localpassword`

4. Create a dashboard with these starter cards:

- Total accounts: `SELECT COUNT(*) AS total_accounts FROM accounts;`
- Total balances: `SELECT COALESCE(SUM(balance), 0) AS total_balance FROM accounts;`
- Payments by status:

```sql
SELECT status, COUNT(*) AS payment_count
FROM payments
GROUP BY status
ORDER BY payment_count DESC;
```

- Daily payment volume:

```sql
SELECT DATE(created_at) AS day, COALESCE(SUM(amount), 0) AS total_amount
FROM payments
GROUP BY DATE(created_at)
ORDER BY day;
```

- Statements generated by period:

```sql
SELECT period, COUNT(*) AS statements_generated
FROM statements
GROUP BY period
ORDER BY period DESC;
```

## Legacy Infrastructure Assets

Terraform and AWS-oriented files remain in the repository for historical/reference use, but are not part of the active local run path.

### Terraform Commands (Legacy)

For local Terraform development with LocalStack:

```bash
cd terraform
terraform init -reconfigure
terraform validate
terraform plan -var-file="localstack.tfvars" -var="db_password=localpassword"
terraform apply -var-file="localstack.tfvars" -var="db_password=localpassword"
```

To clean up LocalStack-provisioned Terraform resources:

```bash
cd terraform
terraform destroy -var-file="localstack.tfvars" -var="db_password=localpassword"
```

For AWS-backed deployment (non-LocalStack), use environment-specific values and credentials:

```bash
cd terraform
terraform init
terraform apply -var="environment=dev" -var="db_password=<strong-password>"
```

### LocalStack Pro Compliance Analytics Setup (Athena + CloudWatch)

1. Start LocalStack Pro and export test credentials:

```powershell
$env:AWS_ACCESS_KEY_ID = "test"
$env:AWS_SECRET_ACCESS_KEY = "test"
$env:AWS_DEFAULT_REGION = "ap-southeast-1"
```

2. Apply Terraform resources to LocalStack:

```powershell
cd terraform
terraform init -reconfigure
terraform apply -var-file="localstack.tfvars" -var="db_password=localpassword"
```
3. Start local PostgreSQL and load mock data:

```powershell
./scripts/start-local-postgres-and-load-data.ps1
```

This starts the local `postgres` container and seeds dashboard-ready records into the local `scbbank` database.

4. Sync ledger records from local PostgreSQL into the DynamoDB ledger table:

```powershell
./scripts/sync-ledger-to-dynamodb.ps1
```

```bash
./scripts/sync-ledger-to-dynamodb.sh
```

5. Publish a compliance metric snapshot from local PostgreSQL into CloudWatch Logs and S3 (for Athena):

```powershell
./scripts/publish-localstack-compliance-metrics.ps1
```

```bash
./scripts/publish-localstack-compliance-metrics.sh
```

6. Verify resources:

```powershell
cd terraform
terraform output athena_workgroup_name
terraform output athena_compliance_database_name
terraform output compliance_dashboard_name
```

Notes:

- The ledger sync script reads from the backend postgres container and writes items to DynamoDB table `sbcbank-dev-ledger` by default.
- Athena dataset source bucket and query results bucket are provisioned by Terraform.
- CloudWatch dashboard widgets and alarms read custom metrics under namespace `SBCBank/Compliance`.
- Run the publish script whenever you want fresh compliance KPI values reflected in CloudWatch and Athena.
